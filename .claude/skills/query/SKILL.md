# query 技能

## 工作流（严格三步）

### 1. 先读 index
打开 `wiki/index.md`，按用户问题中的关键词和分类（sources / entities / concepts / syntheses）筛出**候选页面 3-7 个**。

### 2. 深度阅读相关页
对每个候选页面执行 Read：
- 优先读 `synthesis` 类（已编译过的综述）
- 其次读直接命中关键词的 `concept` / `entity`
- 通过 `## 关联连接` 顺藤摸瓜，最多两跳

### 3. 综合回答
- **必须用相对链接引用来源**：`[标题](category/标题.md)`
- 回答末尾列出"引用页面"清单
- 数字 / 结论必须标注来自哪一页
- 矛盾结论要并列展示，不可只挑一种

## 兜底规则

如果 index 和深度阅读后仍没有相关内容：

> 本地知识库中未找到相关结论，以下为通用知识回答：
> ...

**禁止**：
- 凭模型记忆直接回答业务问题
- 编造 wiki 中不存在的页面或结论
- 引用没有读过的页面

## 输出模板

```markdown
## 回答

[正文，引用必须用 [标题](path) 格式]

## 引用页面
- [页面1](path1.md)
- [页面2](path2.md)
```
