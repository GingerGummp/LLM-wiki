01 为什么再多 Prompt 都救不了你

Doc 收藏夹 200+，写周报还是从零搜起
在 Doc 系统里搜「印尼 金币 上限」，30 条结果翻不到那次实验结论
AI 答案都很通用，因为它根本不懂你的业务
ChatGPT 不知道你公司的指标口径、历史决定、内部黑话
试过 Notion AI / 通用 Doc 智能助手，没真正解决
检索范围太杂、命中率低、无法 fork / 定制
KEY INSIGHT

AI 提效的真正瓶颈，不是 Prompt 写得好不好，而是有没有一份结构化、可被 AI 高效检索、双向链接、持续更新的私有知识库。

02 方案：LLM-Wiki 四层目录契约

参考 Karpathy LLM-Wiki 规范，落到本地 Markdown + Claude Code 环境：

# 三大不变量
1. raw/ 绝对只读 — 唯一真相来源
2. 强制双链 — 每页 ## 关联连接，禁止孤岛
3. 冲突共存 — 不静默覆盖，开 ## 知识冲突 区块

03 四个可借鉴的核心组件

COMPONENT 01

LLM-Wiki Schema

四层目录契约 + Frontmatter 规范 + 双链不变量。写进 CLAUDE.md，Claude Code 启动自动加载。
  • raw / inbox / project / wiki 四层
  • concepts / entities / sources / syntheses 四类
  • index.md + log.md 必备基石
COMPONENT 02

Skill 四件套

把 Claude Code 变成知识库专属编辑器。/bootstrap /ingest /query /lint 覆盖「迁移→摄入→检索→维护」全链路。
  • /bootstrap — 一键从云文档父链接批量初始化
  • /ingest — 把碎片摄入并自动建双链
  • /query — 在 wiki 中检索并强制引用
  • /lint — 死链 / 游离 / 索引 / 冲突
COMPONENT 03

Agent 工作流体系

7 个 Prompt Agent，覆盖高频任务：业务总览 / 实验设计 / 数据分析 / 论文调研 / Prompt 优化 / 周报 / 复盘。
  • 把模糊需求转结构化 A/B 八件套
  • 一篇 paper 出 5 条业务可迁移假设
  • Owner 视角五段式周报
COMPONENT 04

MCP 插件拼装

让 AI 直接读公司内部系统。abtest / kwaibi / docs / kconf / webshell 等内部 MCP 接入示例。
  • abtest-mcp — 直接跑 T-test 出结论
  • kwaibi-mcp — Hive SQL 自动执行
  • docs-mcp — 读写公司 Doc

04 四大 Skill 详解

Skill 角色 触发 写权限
/bootstrap 一键迁移 从云文档父链接批量初始化 wiki 仅写 raw/ + wiki/
/ingest 编译器 摄入 raw / inbox 到 wiki 仅写 wiki/
/query 检索器 在 wiki 中查询并引用回答 只读
/lint 健康度扫描 死链 / 游离 / 索引 / 冲突 报告，需用户确认才修复

四个 Skill 各司其职：bootstrap 解决冷启动；ingest 日常增量；query 让 AI 真正用上知识库；lint 防熵增。

07 适合谁 / 不适合谁

✓ 适合

  • 长期 owner 一块业务、需要积累的人
  • 写代码 / 做研究 / 做策略，希望 AI 真正帮你做判断
  • 愿意每周维护 20 分钟的人

✗ 不适合

  • 一次性小项目、一周后不再回头
  • 不愿意维护目录、不写双链的人
  • 强依赖跨部门协作、内容必须留在公司内部 Doc 系统
